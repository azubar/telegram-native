# Керівництво з використання компоненти TelegramNative в 1С:Підприємство / BAS

`TelegramNative` — це високопродуктивна зовнішня компонента Native API для платформи 1С:Підприємство / BAS, написана мовою **Rust** з використанням офіційної бібліотеки **TDLib (Telegram Database Library)**.

## 🚀 Переваги компоненти
- **Безпека пам'яті та швидкість**: Реалізація на мові Rust гарантує відсутність витоків пам'яті та максимальну швидкість обробки даних.
- **Кросплатформеність**: Підтримка Windows (x86, x64) та Linux (x86, x64).
- **Повна підтримка TDLib JSON API**: Можливість виконувати будь-які запити Telegram API (боти, клієнтські акаунти, канали, групи, файли).
- **Асинхронний режим**: Фонове отримання оновлень від Telegram без блокування інтерфейсу користувача.

## 📦 Встановлення та підключення

### Варіант 1. З макету конфігурації (Рекомендовано)
1. Завантажте файл релізу `AddIn.zip`.
2. Додайте його в конфігурацію як Общий Макет з типом **Двоичные данные**.
3. Підключіть компоненту в коді BSL:

```bsl
НазваМакету = "ОбщийМакет.TelegramNative";
Ім'яКомпоненти = "TelegramNative";

Якщо Не ПодключитьВнешнююКомпоненту(НазваМакету, Ім'яКомпоненти, ТипВнешнейКомпоненты.Native) Тоді
    ВызватьИсключение "Не вдалося подключити зовнішню компоненту TelegramNative!";
КонецЕсли;

Телеграм = Новый("AddIn." + Ім'яКомпоненти + ".TelegramNative");
```

### Варіант 2. З файлу на диску

```bsl
ПутьКАрхиву = "C:\AddIns\AddIn.zip";
Ім'яКомпоненти = "TelegramNative";

Якщо Не ПодключитьВнешнююКомпоненту(ПутьКАрхиву, Ім'яКомпоненти, ТипВнешнейКомпоненты.Native) Тоді
    ВызватьИсключение "Помилка підключення компоненти TelegramNative з диска!";
КонецЕсли;

Телеграм = Новый("AddIn." + Ім'яКомпоненти + ".TelegramNative");
```

## 🛠 Властивості та методи

### Властивості

| :--- | :--- | :--- | :--- | :--- |
| `EventSourceName` | `ИмяИсточникаСобытий` | Рядок | Читання/Запис | Ім'я джерела події, яке передається в 1С при генерації зовнішньої події. За замовчуванням `"TelegramNative"`. |

### Методи

#### 1. `Execute(RequestJson)` / `Выполнить(ЗапросJSON)`
Синхронно виконує запит TDLib, який не вимагає мережевої взаємодії (наприклад, оцінка тексту, встановлення параметрів логування тощо).
- **Аргументи**: `RequestJson` (Рядок) — JSON-запит TDLib.
- **Повертає**: Рядок (JSON-відповідь).

#### 2. `Send(RequestJson)` / `Отправить(ЗапросJSON)`
Асинхронно надсилає JSON-запит до клієнта TDLib. Відповідь надійде асинхронно через метод `Receive()` або як зовнішня подія в 1С.
- **Аргументи**: `RequestJson` (Рядок) — JSON-запит.
- **Повертає**: Не має повертаного значення (Процедура).

#### 3. `Receive()` / `Получить()`
Отримує наступне повідомлення/оновлення від TDLib із очікуванням до 1 секунди.
- **Аргументи**: Немає.
- **Повертає**: Рядок (JSON з оновленням або порожній рядок при таймауті).

#### 4. `SetAsyncMode(Enable)` / `УстановитьАсинхронныйРежим(Включить)`
Вмикає або вимикає фоновий потік отримання оновлень Telegram.
- **Аргументи**: `Enable` (Булево) — `Истина` для запуску, `Ложь` для зупинки.
- **Повертає**: Не має повертаного значення (Процедура).

#### 5. `SetLogFilePath(FilePath)` / `УстановитьФайлЖурнала(ПутьКФайлу)`
Встановлює шлях до файлу логів TDLib.
- **Аргументи**: `FilePath` (Рядок) — шлях до файлу.
- **Повертає**: Булево (`Истина` при успіху).

#### 6. `SetLogMaxFileSize(SizeBytes)` / `УстановитьМаксимальныйРазмерФайлаЖурнала(РазмерФайла)`
Встановлює максимальний розмір файлу логів у байтах.
- **Аргументи**: `SizeBytes` (Число).

#### 7. `SetLogVerbosityLevel(Level)` / `УстановитьУровеньДетализацииЖурнала(Уровень)`
Встановлює рівень деталізації логування TDLib (0 — вимкнено, 1 — помилки, 2 — попередження, 5+ — повне відлагодження).
- **Аргументи**: `Level` (Число).

## ⚡ Режими роботи: Синхронний та Асинхронний

### Синхронний режим (для регламентних завдань)
У цьому режимі 1С самостійно викликає метод `Receive()` у циклі або за розкладом:

```bsl
// Відправка запиту
ЗапросJSON = "{""@type"":""getMe""}";
Телеграм.Отправить(ЗапросJSON);

// Отримання відповіді
ОтветJSON = Телеграм.Получить();
Если Не ЗначениеЗаполнено(ОтветJSON) Тогда
    // Обробка порожньої відповіді або таймауту
КонецЕсли;
```

---

### Асинхронний режим (для керованих форм та клієнтських додатків)
При включенні асинхронного режиму компонента генерує зовнішні події 1С `ОбработкаВнешнегоСобытия`.

```bsl
&НаКлиенте
Процедура ПриОткрытии(Отказ)
    Телеграм.ИмяИсточникаСобытий = "TelegramNative";
    Телеграм.УстановитьАсинхронныйРежим(Истина);
КонецПроцедуры

&НаКлиенте
Процедура ОбработкаВнешнегоСобытия(Источник, Событие, Данные)
    Если Источник = "TelegramNative" И Событие = "Response" Тогда
        // Данные містять JSON-рядок від відповіді Telegram TDLib
        ОбработатьОтветTelegram(Данные);
    КонецЕсли;
КонецПроцедуры

&НаКлиенте
Процедура ПриЗакрытии(ЗавершениеРаботы)
    Телеграм.УстановитьАсинхронныйРежим(Ложь);
КонецПроцедуры
```

## 🔐 Приклад ініціалізації та авторизації

Низькорівнева ініціалізація авторизації бота в Telegram через TDLib:

```bsl
// 1. Встановлення параметрів TDLib
ПараметрыTDLib = "{"
    + """@type"":""setTdlibParameters"","
    + """parameters"":{"
    + """api_id"":123456,"
    + """api_hash"":""your_api_hash_here"","
    + """system_language_code"":""uk"","
    + """device_model"":""1C:Enterprise"","
    + """system_version"":""8.3"","
    + """application_version"":""1.8.65"","
    + """database_directory"":""C:/TDLibData"","
    + """use_message_database"":true"
    + "}"
    + "}";

Телеграм.Отправить(ПараметрыTDLib);

// 2. Авторизація за допомогою токена бота
АвторизацияБота = "{"
    + """@type"":""checkAuthenticationBotToken"","
    + """token"":""123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ"""
    + "}";

Телеграм.Отправить(АвторизацияБота);
```

## 📝 Налаштування логування

Для діагностики та пошуку помилок можна увімкнути детальний журнал TDLib:

```bsl
Телеграм.УстановитьУровеньДетализацииЖурнала(2); // 2 = попередження та помилки
Телеграм.УстановитьМаксимальныйРазмерФайлаЖурнала(10 * 1024 * 1024); // 10 МБ
Телеграм.УстановитьФайлЖурнала("C:\Logs\tdlib.log");
```
